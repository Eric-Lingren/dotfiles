## Trace

### Design

Read existing scripts for style conventions (validate-schema.py, validate_registry.py):
argparse-based CLI, plain functions, print to stderr on error + sys.exit(1). Read the
seed file docs/seeds/20260709-1616-improve-directory-structure-skill.json for the full
scanner architecture decisions (core vs pluggable extractors, litmus set, --check mode
deferred to a separate scanner-level task, not this one).

Designed input contract:
- nodes: {id, path, type, cluster, is_shared?, content_signature?}
- edges: directed {from, to, kind?} meaning "from depends on to"
- metrics: keyed by node id: {fan_in, fan_out, dir_depth, orphan, churn?}

Designed output contract: {schema_version, generated_at, summary, findings[]} where each
finding has {id, principle, title, nodes, evidence, detail}.

Mapped each litmus item to a detector function:
- ADP: DFS-based cycle detection with back-edge reconstruction + cycle dedup by
  rotating to canonical start.
- SRP: node fan-out >= threshold spanning >= N distinct target clusters.
- CCP (locality): is_shared node, fan_in >= 2, all incoming edges from one cluster.
- CRP: is_shared node, fan_in == 1 (mutually exclusive with CCP via fan_in branch).
- DRY: pairwise Jaccard similarity over content_signature (fallback: out-edge target
  sets) >= threshold.
- SDP: node with explicit metrics.churn >= threshold AND fan_in >= threshold.
- MODULARITY: per-cluster internal/touching edge ratio < threshold, gated by a
  minimum touching-edge count to avoid flagging sparse clusters.

### Implementation

Wrote claude-code-shared/scripts/dependency-graph-findings.py:
- load_graph(): reads file arg or stdin, validates nodes have unique ids and edges
  reference known node ids, exits 1 with stderr message on any malformed input.
- build_indexes(), node_metric(), edge_ref() helpers.
- detect_adp / detect_srp / detect_ccp_and_crp / detect_dry / detect_sdp /
  detect_modularity, each returning finding dicts citing principle + evidence.
- assemble_findings(): runs all detectors, sorts findings deterministically by
  (principle, nodes, title), assigns sequential F0001.. ids.
- CLI via argparse: positional input path (optional, else stdin), --out, and one
  threshold flag per detector (srp-min-fanout, srp-min-clusters, dry-min-similarity,
  sdp-min-churn, sdp-min-fanin, modularity-min-edges, modularity-max-ratio).

No extractor/repo-walking logic included, per task scope -- this is purely the
findings core over an already-built graph.

### Manual verification (per infra no-unit-test convention)

Hand-built fixture: claude-code-shared/scripts/dependency-graph-findings.example.json
- Cycle (ADP): a1 -> a2 -> a3 -> a1, cluster "core", fan-out 1 each (below SRP
  threshold, no other principle collision).
- DRY duplicate: d1/d2 in cluster "core" with identical content_signature token sets
  (jaccard 1.0, above 0.8 threshold).
- Misplaced resource (CRP): r1 marked is_shared in cluster "shared", single consumer
  edge from owner1 (cluster "ownerA") -> fan_in == 1 triggers CRP, not CCP (CCP
  requires fan_in >= 2 by design, so the two never collide on one node).
- Verified no other detector misfires: max fan-out is 1 (SRP needs >=4), no node
  defines metrics.churn (SDP skips nodes without an explicit churn key), and every
  cluster's touching-edge count is below the modularity min-edges gate (6), so
  MODULARITY does not fire on this small fixture.

  python3 claude-code-shared/scripts/dependency-graph-findings.py \
    claude-code-shared/scripts/dependency-graph-findings.example.json

duplicate} -- matches acceptance criterion 2 exactly.

  same 3 findings.
- Malformed input handling: dangling edge reference -> stderr error + exit 1;

- `python3 -m py_compile` on the script -> compiles clean.
