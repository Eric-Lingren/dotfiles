## Trace

### Design

Read claude-code-shared/scripts/dependency-graph-findings.py (T-0077) end to end to
learn the exact input contract:
- nodes: {id, path, type, cluster, is_shared?, content_signature?}
- edges: directed {from, to, kind?}, "from depends on to"
- metrics: keyed by node id: {fan_in, fan_out, dir_depth, orphan, churn?}
Extractors are expected to supply fan_in/fan_out/dir_depth/orphan directly (core only
falls back to deriving fan_in/fan_out from edges when missing).

Read validate_registry.py for style precedent (argparse, default base-dir = parent of
script, stderr+exit(1) on error).

Surveyed the real repo artifacts to ground the design:
- `claude-code-shared/skills/*/SKILL.md` (29 skill dirs)
- `agents/registry.json` (18 agent entries, each with name/file/model/consumers)
- `skill-pipeline.json` (skills[].next[] DAG)
- `resources/model-tiers.json` (skills/agents maps keyed by name)
- `scripts/` (46 files, incl. subdirs architecture-skill-audit/, tooling-detection/)
Confirmed via grep that `subagent_type: <name>` / `subagent_type="<name>"` and
`~/.dotfiles/claude-code-shared/scripts/<path>` are the real literal patterns used in
skill/agent markdown (e.g. skills/build-code/SKILL.md:108
`Agent(subagent_type="build-runner", ...)`, skills/to-tasks/SKILL.md:214 references to
`scripts/next-task-id.sh`).

Noticed agents/registry.json's browser-checker/lint-runner/test-runner consumers still
list a stale skill name `run-tasks` (renamed to `build-code`/`dispatch-tasks` at some
point) that no longer exists as a skill directory on disk -- decided to silently skip
edges to unresolved consumer names rather than error, since dangling-reference
detection is explicitly scoped to the later --check task (T-0080), not this one.

Mapped the task's five named edge-producing signals 1:1 onto edge `kind` values:
1. registry.json consumers -> `registry_consumer` (skill -> agent)
2. subagent_type= spawns -> `spawns` (owning skill/agent -> spawned agent)
3. absolute script-path references -> `script_ref` (owning skill/agent -> script)
4. model-tiers.json skills/agents keys -> `tiered_by` (skill/agent -> model-tiers.json resource node)
5. skill-pipeline.json next[] -> `pipeline_next` (skill -> skill)

Node design:
- skill nodes: cluster = own name, is_shared=false (a skill owns itself).
- agent nodes: cluster="agents", is_shared=true (agents live in a shared top-level
  bucket regardless of consumer count) -- this is what makes CRP/CCP findings fire
  meaningfully for real agents (e.g. single-consumer agents).
- script nodes: cluster="scripts", is_shared=true, one per file under scripts/
  (excludes __pycache__/dot-dirs).
- resource nodes: agents/registry.json, resources/model-tiers.json,
  skill-pipeline.json, resources/repo-policy.json, resources/task-routing.json (only
  those present on disk), cluster="resources", is_shared=true.

churn: best-effort via a single `git log --format= --name-only` call, counted per
relpath. Optional field (SDP skips nodes without it) -- included since it's cheap and
grounds SDP too.

### Implementation

Wrote claude-code-shared/scripts/claude_tooling_extractor.py:
- discover_skills(), load_registry(), discover_scripts(), discover_markdown_files()
- owning_node_id(): maps a repo-relative markdown file path back to skill:<name> or
  agent:<name> via registry file-path match.
- resolve_script_path(): resolves grep-captured script path candidates against known
  script node ids, stripping regex over-capture trailing punctuation.
- compute_churn(): best-effort git log --name-only counter, returns {} on any failure.
- build_graph(base_dir): assembles nodes/edges per the design above, computes
  fan_in/fan_out/dir_depth/orphan/churn metrics, returns {nodes, edges, metrics}.
- CLI: positional base_dir (default: parent of this script), --out.

Wrote claude-code-shared/scripts/scanner.py:
- Thin CLI dispatcher: EXTRACTORS = {"claude-tooling": claude_tooling_extractor.build_graph}.
- --extractor <name> [root] [--out FILE], --list-extractors.
- default_root_for("claude-tooling") = the claude-code-shared repo this script lives
  in (matches the acceptance criterion's literal invocation `scanner --extractor
  claude-tooling` with no root arg).
- Does not compute findings -- documented in the module docstring to pipe stdout into
  dependency-graph-findings.py.
